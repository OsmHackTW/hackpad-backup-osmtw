增加水點資料的解法(目前有600多筆資料，目標是十萬筆)

  • 問機關
    • 教育部：函轉？教育部不鳥我們
    • 國發會研考處：「我們鼓勵政府開放生活資訊資料..」
    • 台北市資訊局：...
  • 分區的指標：
    • 都會區：例如以 500 公尺半徑至少有一台飲水機，計算都會區範圍應該要有多少台，對比現況計算到達率，甚至計算覆蓋率、佔有率...
    • 風景區：遊客中心...
    • 登山區：...
  • 要跟國發會講，生活資訊資訊應該開放

使用者標注

  • 把飲水地圖模式應用到 Umap 上，成為「主題地圖」產製
  • 把 Note 拉出來變成標注
    • Note 很難解
  • 加水點表單的問題
    • 因為定位準確度會差很多，有時是以公里計，應該 zoom 值不夠大的不能點，說明「請放大到 XXX」
  • 資料檢證角色層級
    • 最終防線：OSM 資深圖客
    • 中長期培訓：解 note 志工，配合 policy, guideline, reference
  • 資料品質的標準／準則（稍後另開一個 pad）
    • 熱／冰水不能標錯
    • 位置 + 說明的誤差可以接受到多少？
      • 例如一種參考標準：到達這個點位之後，「五分鐘」之內透過說明內容，使用者可以真的找到該飲水機面前並使用它
      • 考量「無障礙者」也可以無礙使用...
      • 關於尺寸、高度... 透過「型號」來對照設備規格來確認
    • 樓層一定要記得填，一樓也要填
  • 加一個錯誤回報功能，加在水點跳出的內容裡
