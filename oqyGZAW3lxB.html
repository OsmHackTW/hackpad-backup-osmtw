文案

簡介

OpenStreetMap是一個開放的線上地理資料庫，由於其開放授權、向量資料、即時更新的特性，能夠自由運用在各種不同領域，如手機、導航裝置上的離線圖資，方便出國玩省資費，或是登山健行導航。在多次重大災害中，OpenStreetMap發揮其即時更新的強項，防災社群HOT與衛星圖資商合作，快速在災難發生的短短幾天時間，號召分佈全球各地的繪圖志工，畫出最新的災區地圖，提供進駐災區的救難隊使用。

問題

  • 還不知道OSM的訪客
    • 這是台灣的計畫還是全世界性的?
      • 後者
    • OSM 是向量地圖嗎？
      •  2016 年最常被問到的問題
      • OSM 不只是地圖，是地理資料庫，呈現的地圖有點陣圖磚也有向量圖磚(mapbox studio, maps.me)
    • 這跟Google Map 有何不同?
    • 為什麼不用 Google Map?
      • (必答，請做好準備，很重要不必說三次)
      • 可參考 wiki 上的常見問題
    • 話術
      • 介紹時統一口徑，第一句先說明OSM是地理資料庫，而不是以地圖為中心來介紹下去( Yellowsoar 的 Idea )
    • 如果有可以先寫好話術會更容易同步
  • 技術需求的訪客
    • 有 API 可以用嗎?
      • 前端 
        • OSM api
        • OverPass api
        • overpass turbo QL
    • 技術名詞字典
      • 座標系統：介紹什麼是 WGS 84 (EPSG 4326)、TWD 97 (EPSG 3826)、EPSG 900913
  • 需要提到分帶嗎？（國內有不少機關很愛用啊...）
  • 可以做成活頁夾的形式，或是維基百科的 QR Code
    • 座標轉換：座標轉換工具、投影法
    • 圖磚輸出程式：mapnik mapsforge
    • 幾何資料格式：shp osm pbf geojson
    • 幾何運算工具：PostGIS
    • 常用 POI 表達方式：
    • 專有名詞提供維基百科或 OSM Wiki 的 QR Code 讓來賓可以快速閱讀
    • https://www.the-qrcode-generator.com/
