  • 方案二：民眾拍照+圖客志工上資料
    • 基本方式：可以先徵集民眾的照片（提供簡單教學），再靠專業志工上傳資料到地圖上
    • 操作流程
      • 室外水點，拍照依靠 GPS 資訊，需要提醒民眾打開 gps 功能
      • 室內水點，拍照，並紀錄地點地址
      • 上傳照片與位置資訊、備註說明
      • 由專業志工抓出照片經緯度、或從地址轉出經緯度
      • 志工與民眾，依據照片內容、現場勘查，幫忙確認 tag、水點標註位置是否正確
    • 關於「照片」
      • 「照片」需要提供拍照範例，例如「飲水台」、「飲水機」等，原則上整個機體盡量呈現，單點多拍
      • 可以發起「我與飲水機自拍」、「錯位拍法」等企劃方式
      • 若想要在地圖水點上同時呈現出照片，則：
        • 建置相簿網站 picasa 
        • cc 授權
        • 運用「url？」
          • 用image=
          • http://wiki.openstreetmap.org/wiki/Historical_Objects/Popup#Bildvorschau
          • http://wiki.openstreetmap.org/wiki/Key:image
    • 現勘回報
      • 在 osm 介面右下角，可以新增 note 拖曳到要提報的位置，可以填寫內容文字，這也是一種飲水使用者回報資訊與現況勘查的方式

電腦操作簡單步驟小記（依照順序）

  • 類型一：飲水機
    •

    申請帳號

    先登入

    按編輯

    point （注意！點，不要加在線條上：例如道路、河川、電線、捷運！）

    amenity=drinking_water

    name 名字=「飲水型態」（簡化，例如飲水台、飲水機、奉茶）

    tag

    alt_name「地名＋飲水型態」（完整名稱，飲水地圖使用上就用此完整名稱）

    level

    indoor

    brand「『賀眾牌』、3M」

    bottle

    source「若是賀眾提供的資料，則填入賀眾公司全名」

    description

    冷熱水（避免與淋浴用水混淆，所以都用 drinking_water:）

    • drinking_water:hot_water=yes
    • drinking_water:warm_water=yes
    • drinking_water:cold_water=yes（意思是冷水！並不是冰水）

    note 寫給其他地圖編輯者看的

    fixme=yes 若需要其他人幫忙修正

    按下儲存！

    若要放該水點的圖片則是要放到...?

    類別一：直接標示飲水機/點 標法與 tag 說明

    • name（「地標」+「飲水型態」，若無名字則填入「飲水點」或是把 tag 撈回來組成 name）
      • 飲水型態：飲水機、直飲台、生飲台（沒有濾心）
      • 加水站建議用 amenity=water_point（對象不是人）
      • 舉例：萬隆捷運站生飲台、警察局飲水機
    • amenity=drinking_water 用途、做什麼用，單獨標示飲水機用的，捷運站內的飲水機
      • http://wiki.openstreetmap.org/wiki/Tag:amenity%3Ddrinking_water
    • description 描述
      • 使用者自填 ex山區「最後水源」
      • 是否需要自備容器？重點在想描述出, 可"直接用嘴喝", 不需任何杯瓶，幾種中文分法：
        • 需自備容器/不需自備容器
        • 備容/免容
        • 免容器
        • 直接用嘴喝
    • source 資料來源
      • survey 民眾自己提的
    • indoor=yes http://wiki.openstreetmap.org/wiki/Key:indoor
      • level=樓層（數字）http://wiki.openstreetmap.org/wiki/Key:level
      • access=* （例如環資屬於 permissive）
      • opening_hours 開放時間 http://wiki.openstreetmap.org/wiki/Key:opening_hours
    • brand（ex賀眾, 3M）
    • 冷溫熱
      • drinking_water:hot_water=yes
      • drinking_water:warm_water=yes
      • drinking_water:cold_water=yes（意思是冷水！並不是冰水）
    • bottle=yes（意思是可以很容易用瓶水裝水）
      • bottle=require（可以考慮新增）
      • https://taginfo.openstreetmap.org/keys/bottle#values

    類別二：標示該建築物/區域有提供飲用水/機 標法與 tag 說明

    • name
      • 舉例：台灣環境資訊協會
    • drinking_water=yes/或其它（用在建築物內的飲水機）
    • 這類飲水點附屬於機關內、裝水者需要詢問場地機關、可能地標物本身有原先用途（如店家、機構）而飲水設施只是附屬
      • http://wiki.openstreetmap.org/wiki/Key:drinking_water
    • source 資料來源 (這個類別，有辦法撰寫「資料來源」嗎？)
    • description 描述 (這個類別，有辦法撰寫「描述」嗎？)
    • drinking_water:hot_water=yes
    • drinking_water:warm_water=yes
    • drinking_water:cold_water=yes
    • drinking_water:bottle=yes
    • indoor=yes http://wiki.openstreetmap.org/wiki/Key:indoor
      • level=樓層（數字）
      • access=* （例如環資屬於 permissive）
      • opening_hours 開放時間 http://wiki.openstreetmap.org/wiki/Key:opening_hours

    其他討論

    • 賀眾的資料若是地址則是自己轉。
    • 相機開 GPS 位置，在電腦上面看位置，依照位置定上去。
    • OSMand 打開OSM Editing plugin，可以添加商店資訊
    • 有一位「設計師」可能有興趣協助
    • 可以先翻譯 drinking water 頁面中文化
    • 詢問「天龍國單車路網」的作者，因為功能相近，社團內可聯繫到
    • 若是已經有「地址」資料要標註在osm編輯介面上，操作程序為：
      • 進入 TGOS 網頁
      • 輸入「地址」，記得要有「區」
      • 按下轉換
      • 在頁面上取得座標數字XY
      • 要把座標用線上版的轉換方式
      • 選擇 TWD97 二度分帶，輸入座標數字XY
      • 取得 WGS84 的座標數字
      • 輸入到OSM網址上，只能五位數，把逗號換成斜線
      • 這樣網頁中的標註點就出現在適當的位置了，可以進一步開始輸入名稱與 tag
    • 這部份或許可以寫一些計算服務
      • TGOS 有 API 可以申請使用
    • 運用 xml Jason ...? 一次把地址處理
      • 然後請志工現勘水點並修正
    • 對於 tag 用法，可至 taginfo 查詢
