土地測量、基石、三角點等標注方式

標注方式

山頂與基石應該分別標注。

Pro

  • 山頭應標記在最高的那塊非人造非生物的物體座標 。
  • 因為一個山頭有時好幾個基石。
  • 不同命名方便用戶遍識，如石門山目前共用基石、山頭。山頭仍為「石門山」、基石改為「石門山三等三角點」容易遍識。 兩個獨立 "object" 讓搜尋引擎索引的時候可以列出兩筆結果。名稱不同，對使用者可用性比較高，否則就得在操作界面上提示不同的圖徵。 
  • 由於 "peak" 不易精確，同一個山頂不應該超過一個 peak.  但 survey point 應該要能精確.  
  • peak 很容易被 mapper 移來移去, 但理論上 survey point 應該小心測量, 分成兩個比較不會被弄壞 

Con

  • 直接利用三角點當作不同比例下的最高顯示點，可以免去程式計算的複雜度。 或者迫使現有地圖界面顯示出三角點
  • 在小畫面的導航器上，三角點或山點可能互相干擾覆蓋。

    標注需求

    • 區分基石類型
    • 需要提供基石編號

    標示範例

    • man_made=survey_point // 人工基石

    • checkpoint=hiking // 打卡點
    • checkpoint:type=survey_point
    • name=* // 名稱
    • name:zh=* // 中文名稱
    • name:en= // 英文名稱
    • alt_name=* // 其他名稱
    • ele=* // 海拔
    • ref:survey_point= // 參考
